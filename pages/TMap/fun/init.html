# 地图初始化渲染

刚刚我们已经初步学习了创建一个地图，这一节我们深入了解初始化地图

首先，还是渲染一个地图，不过这次我们额外增加一些配置

    import { initMap, setMapContainer } from "../js/TMap.js";
    const option = {
      targetContainerId: "mapContainer1",
      centerLng: 104.076693,
      centerLat: 30.513315,
      zoom: 13,
      minZoom: 10,
      maxZoom: 16,
      // baseMap: "dark",
      controls: ["Zoom", "OverviewMap"],
      isAllowMouseWheelZoom: false,
      isAllowDoubleClickZoom: false,
    };
    const { TMap, baseLayer } = initMap(option);

baseMap设置为dark加载夜景地图，但是我们注释掉了，加载了默认的OSM地图

controls字段为我们初始化添加了两个控件

isAllowDoubleClickZoom禁用了地图原生的双击缩放交互

isAllowMouseWheelZoom禁用了地图原生的鼠标滚轮缩放交互

minZoom与maxZoom为我们限制可缩放的层级

当然初始化地图不止这些可配置项，通过API可以解更多初始化地图的配置，

初始化地图所调用的api返回了一个对象，我们可以解构得到TMap,baseLayer这两个值

TMap为我们所渲染的地图对象（实质为ol/Map类），而baseLayer为地图的底图图层（实质为ol/Layer类）

记住这两个值，在我们后续调用其他方法的过程中十分重要

我们的开发过程中若是地图容器宽高不固定或者使用弹性布局可能会出现地图canvas没有占满容器或者超出容器的情况，这是为什么呢地图视图并非随着容器大小实时更新，当容器状态发生变化时我们需要调用resizeMap来强制缩放地图，使地图容器大小与地图画布相匹配。可能熟悉eCharts的同学对这个方法有种熟悉的感觉，确实，这个方法和eCharts的resize()方法原理类似，解决的问题也是一致的，都是为了实现canvas相对于容器大小的强制更新

# 改变地图容器

上一节是把地图容器渲染到一个id为container1的容器中，如果我们想把地图挪到另一个容器且保持原样，但不想重新加载，我们该怎么做呢？

setMapContainer可完美解决这一问题

    let container = "mapContainer1";
    document.getElementById("setContainerBtn").onclick = () => {
      container =
        container === "mapContainer1" ? "mapContainer2" : "mapContainer1";
      setMapContainer(TMap, container);
    };

观察上述代码，除了普通js逻辑外，我们调用了这个方法setMapContainer，所传的第一个参数为刚刚初始化返回的TMap，第二个参数便是所要改变的容器id

ps:

这就是函数式编程的特殊之处了，一切皆参数，直接将地图实例传入函数，然后函数内部进行对地图实例的的操作，我们TMap中的API都是遵循这种方式，需要操作的对象实例当做参数传进去，TMap就是一个我们API中十分常见的参数

# 改变地图底图

要是实现一个类似于百度高德地图那样切换矢量/影像地图显示该怎么做呢，要是不了解gis概念，可能只是简单理解重新渲染了个地图在上面，但是重新渲染在实际场景中是不切实际的，正确做法应该是从图层出发，改变地图底图图层对应的数据源Source来实现这个功能

尝试使用setBaseMap来实现这个功能

    // 矢量地图按钮
    document.getElementById("vectorBtn").onclick = () => {
      setBaseMap(baseLayer, "vector");
    };
    // 影像地图按钮
    document.getElementById("imageBtn").onclick = () => {
      setBaseMap(baseLayer, "image");
    };
    //夜景地图按钮
    document.getElementById("darkBtn").onclick = () => {
      setBaseMap(baseLayer, "dark");
    };

# 添加/删除控件

在我们开始初始化地图的时候，通过controls的配置初始添加了一些控件在地图上，要是我们在使用过程中想要在添加其他控件呢

可以试试addMapControl

当然了有添加就有删除removeMapControl

我们尝试用这两个实现添加与删除功能

    //添加/删除全屏组件按钮
    let control;
    document.getElementById("addControlBtn").onclick = () => {
      if (control) {
        removeMapControl(TMap, control);
        control = undefined;
      } else {
        control = addMapControl(TMap, "FullScreen");
      }
    };

ps:

控件实际上是可以通过css进行样式修改与覆盖（覆盖需要在ol.css引入后）的，但是实际使用场景可能更多会自己实现一个控件功能而非去修改原生控件样式
