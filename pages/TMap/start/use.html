# 资源组成

使用之前需要简单了解一下相关资源组成

  • ol.js：OpenLayers源JavaScript GIS库文件

    • 无特殊情况请勿修改

  • ol.css：OpenLayers源CSS(层叠样式表)文件

    • 无特殊情况请勿修改

  • TMap.js：TMap方法集合主文件

    • 该文件包含了一系列基于ol实现地图功能的方法并通过export方法导出
    • 掌握ol相关知识后可对此文件进行调整，维护与扩充

  • mapConfig.js:公网环境底图瓦片资源相关配置文件

    • 为了地图底图可配置化，将底图资源配置单独拆离成一个文件
      点击查看配置文件代码书写范例及注释 
      /*eslint-disable*/
window.baseMapConfig = {
  //矢量地图
  vector: {
    //必须 此处键名即为initMap时的baseMap值（string） eg： baseMap: "vector",
    type: "vector", //非必须
    label: "矢量地图", //非必须
    layer: {
      layerType: "XYZ", //非必须
      source: new ol.source.XYZ({
        // url: "http://rt0.map.gtimg.com/realtimerender?z={z}&x={x}&y={-y}&type=vector&style=0", //腾讯矢量
        url: "http://wprd04.is.autonavi.com/appmaptile?lang=zh_cn&size=1&scl=1&style=7&x={x}&y={y}&z={z}", //高德矢量（白底非矢量） （无缺失且加载速度快 scl1含标注2不含标注）
        // url: "http://wprd04.is.autonavi.com/appmaptile?lang=zh_cn&size=1&style=8&x={x}&y={y}&z={z}", //高德矢量 （真实矢量路网地图）
        // url: "https://map.geoq.cn/ArcGIS/rest/services/ChinaOnlineCommunity/MapServer/tile/{z}/{y}/{x}",//arc矢量
        wrapX: false, //是否在X轴方法repeat渲染底图
        //底图颜色
        crossOrigin: "anonymous", //允许跨域请求地图资源
        tileLoadFunction: function (imageTile, src) {
          //瓦片加载函数，用于底色修改
          const img = new Image();
          img.crossOrigin = "";
          img.onload = function () {
            const canvas = document.createElement("canvas");
            const w = img.width;
            const h = img.height;
            canvas.width = w;
            canvas.height = h;
            const context = canvas.getContext("2d");
            context.drawImage(img, 0, 0, w, h, 0, 0, w, h);
            const imageData = context.getImageData(0, 0, w, h);
            for (i = 0; i < imageData.height; i++) {
              for (j = 0; j < imageData.width; j++) {
                const x = i * 4 * imageData.width + j * 4;
                const r = imageData.data[x];
                const g = imageData.data[x + 1];
                const b = imageData.data[x + 2];
                // 高德底图校色
                //校色需要自行通过取色器获取到瓦片本身颜色，在通过if else判断改成你想要的颜色
                if (
                  (r === 252 && g === 249 && b === 242) ||
                  (r === 233 && g === 231 && b === 226) ||
                  (r === 238 && g === 237 && b === 232) ||
                  (r === 238 && g === 237 && b === 231) ||
                  (r === 255 && g === 255 && b === 255)
                ) {
                  imageData.data[x] = 239;
									imageData.data[x + 1] = 243;
									imageData.data[x + 2] = 250;
                } else if (r === 255 && g === 164 && b === 92) {
                  imageData.data[x] = 248;
                  imageData.data[x + 1] = 211;
                  imageData.data[x + 2] = 146;
                }
              }
            }
            context.putImageData(imageData, 0, 0);
            imageTile.getImage().src = canvas.toDataURL("image/png");
          };
          img.src = src;
        },
      }),
    },
  },
  //影像地图
  image: {
    type: "image",
    label: "影像地图",
    layer: {
      layerType: "XYZ",
      source: new ol.source.XYZ({
        // url: "http://server.arcgisonline.com/ArcGIS/rest/services/World_Imagery/MapServer/tile/{z}/{y}/{x}",//arc影像
        url: "https://webst01.is.autonavi.com/appmaptile?style=6&x={x}&y={y}&z={z}", //高德影像
        wrapX: false,
        crossOrigin: "anonymous",
      }),
    },
  },
  //夜景地图
  dark: {
    type: "dark",
    label: "夜景地图",
    layer: {
      layerType: "XYZ",
      source: new ol.source.XYZ({
        url: "http://map.geoq.cn/ArcGIS/rest/services/ChinaOnlineStreetPurplishBlue/MapServer/tile/{z}/{y}/{x}", //ArcGis 深色底图
        wrapX: false,
        //底图颜色
        // crossOrigin: "anonymous",
      }),
    },
  },
  //若需扩充相关配置，在这里添加
};

# 下载

首先需要下载TMap.js的底层支持 ol.js与ol.css，这两个文件为OpenLayers官方下载地址 (opens new window)中的vx.x.x.-package.zip包解压后的静态资源文件

注：

虽然OL也提供npm包，但是由于npm包需要node环境支持以及OL npm包特殊的引入结构，还是直接使用静态文件作为底层支持

除了OL资源外，TMap.js与mapConfig.js与也是必不可少的，下载仓库地址为:下载地址 (opens new window)

注：

仓库中不仅有TMap.js和mapConfig.js文件，还有ol.js与ol.css，因为TMap支持版本因为时间差可能无法及时的与OL最新版本同步，所以建议直接从仓库下载所有相关资源文件（仓库为文档仓库地址，位于仓库tmap-ol-doc/css/和tmap-ol-doc/js/下）

# 引入与使用

拿到资源后我们就可以在项目中引入了

# HTML中引入与使用

普通html中实现地图功能时按如下方式引入

在<head>标签中通过<link>引入ol.css,通过script引入ol.js与mapConfig.js

通过<script type="module">的方式支持import语法，并以如下方式按需引入所需要的方法即可开始使用了







 
 
 


 
 
 




  <!DOCTYPE html>
  <html lang="en">
    <head>
      <meta charset="UTF-8" />
      <meta http-equiv="X-UA-Compatible" content="IE=edge" />
      <meta name="viewport" content="width=device-width, initial-scale=1.0" />
      <link rel="stylesheet" href="../css/ol.css" />
      <script src="../js/ol.js"></script>
      <script src="../js/mapConfig.js"></script>
      <title>Document</title>
    </head>
    <script type="module">
      import { initMap } from "../js/TMap.js";
      const option = {xxx:xxx}
      initMap(option);
    </script>
  </html>

# Vue中引入与使用

vue中引入实际与上述类似，因为vue实际也是一个单页面应用，只有一个html入口

我们首先需要找到public下的index.html








 

 
 














<!DOCTYPE html>
<html lang="">
  <head>
    <meta charset="utf-8" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width,initial-scale=1.0" />
    <link rel="icon" href="<%= BASE_URL %>favicon.ico" />
    <link rel="stylesheet" href="<%= BASE_URL %>ol.css" />
    <title><%= htmlWebpackPlugin.options.title %></title>
    <script src="<%= BASE_URL %>ol.js"></script>
    <script src="<%= BASE_URL %>mapConfig.js"></script>
    <!-- <script src="<%= BASE_URL %><%= VUE_APP_MAP_CONFIG_JS %>"></script> -->
  </head>
  <body>
    <noscript>
      <strong
        >We're sorry but <%= htmlWebpackPlugin.options.title %> doesn't work
        properly without JavaScript enabled. Please enable it to
        continue.</strong
      >
    </noscript>
    <div id="app"></div>
  </body>
</html>

实际与html引入方式相差无几，但是有点需要注意的地方，vue的index.html中需要添加<%= BASE_URL %>基路径，若出现不同环境不同配置的情况，需要自行配置环境变量来加载不同的mapConfig.js文件，至于如何配置vue环境变量，请自行网上查阅，这里不做过多赘述

可能大家已经发现了TMap.js并没有在上述过程中引入，因为import语法我们习惯把TMap.js作为一个公共的方法供全局调用，所以一般放在@/utils/下，这样你就可以像下面这样在你的.vue文件中通过import引入相应的方法


 












import { defineComponent } from "vue";
import {initMap} from "@/utils/TMap.js";

export default defineComponent({
  data() {
    return {
      option:{xxx:xxx}
    };
  },
  mounted(){
    initMap(this.option);
  },
});

# React中引入与使用

React中实际与Vue没什么差别，同样是在public/index.html引入ol.css,ol.js与mapConfig.js

在项目存储公共js方法的地方添加TMap.js

然后在你的项目中通过import使用它吧

因为与vue类似，这里就不做示例书写了，感兴趣的话可以自行研究react中TMap的实际使用

注：

本文档中的后续示例均以普通html为基础书写
